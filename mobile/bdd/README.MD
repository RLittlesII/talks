## Abstract
Behavior is a basic concern for humans.  Behavior Driven Design is an abstract concept with various definitions agreed upon by the community of developers.  The Behavior of our Teams has to change in order to practice Behavior Driven Development (BDD).  Where is the payoff?  Skilling up a team is annoying.  Writing automated tests slows velocity and deliverables.  Legacy concerns become black holes that keep your application stagnant.  We'll look at how Behavior Driven Design and Development can remedy some of these concerns, and reduce friction for others.  While there is no silver bullet, having a plan for how your code interacts, is generally better than not.  BDD can help with evoving designs, ensuring sure objects answer questions, and encapsulating concerns to make boundaries more explicit.  So lets dive into how to change our Behavior so we can Develop easier to reason about software.
